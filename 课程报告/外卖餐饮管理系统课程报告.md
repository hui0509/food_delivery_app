# 外卖餐饮管理系统课程报告

## 📋 项目概述

**项目名称**：外卖餐饮管理系统  
**项目类型**：Web全栈应用  
**技术栈**：Vue.js + Flask + MySQL + Redis  
**开发周期**：2025年11月 - 2026年2月  
**项目规模**：前后端完整实现，包含70+功能模块

## 🎯 项目目标

开发一个功能完整的外卖餐饮管理系统，实现：
- 用户端：浏览店铺、下单、评价、个人中心
- 商家端：店铺管理、菜品管理、订单处理
- 管理员：系统管理、用户管理、配送管理
- 完整的业务流程：浏览→下单→支付→配送→评价

- **店铺表(shop)**：店铺信息、状态管理
- **菜品表(dish)**：菜品详情、价格库存
- **订单表(order)**：订单流程、状态跟踪
- **评价表(review)**：评价内容、评分系统

## 🔧 开发难点与解决方案

### 难点1：跨域问题
**问题**：前端开发服务器与后端API跨域访问
**解决方案**：
- 配置动态CORS，支持多端口访问
- 开发环境宽松配置，生产环境严格限制

### 难点2：身份认证
**问题**：多角色权限管理复杂
**解决方案**：
- JWT Token + 权限装饰器
- 三级权限：用户→商家→管理员

### 难点3：数据一致性
**问题**：订单状态同步困难
**解决方案**：
- 数据库事务保证数据一致性
- Redis缓存减少数据库压力

### 难点4：文件上传
**问题**：图片上传和安全存储
**解决方案**：
- 文件类型限制
- 中文文件名转拼音
- 前端CDN路径映射
